Quickstart: create a production VPS

Recommended workflow: send a POST request, get the machine identifier, then track its status until ready. 

curl -X POST "https://api.novahost.dev/v1/instances" \
  -H "Authorization: Bearer NH_LIVE_xxxxx" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "prod-web-fr-01",
    "plan": "vps-pro-4",
    "region": "fr-par-1",
    "image": "ubuntu-24.04",
    "backups": true,
    "ssh_keys": ["key_main_admin"]
  }'
Expected response
201 Created with instance_id, status = provisioning
Average provisioning time
35 to 60 seconds depending on region
Next step
GET /v1/instances/{instance_id} until status = ready

Authentication

NovaHost utilise un token Bearer par environnement (test ou live).

Environnement Base URL Token prefix
Sandbox https://api-sandbox.novahost.dev NH_TEST_
Production https://api.novahost.dev NH_LIVE_

Never store the token client-side. Use a backend or a secrets manager.

Instance lifecycle

POST/v1/instances

Create a new instance.

GET/v1/instances/{id}

Get real-time status updates.

POST/v1/instances/{id}/actions/reboot

Secure reboot with full traceability.

DELETE/v1/instances/{id}

Controlled deletion with confirmation.

Infrastructure event webhooks

To avoid heavy polling, configure a server-side webhook endpoint. You will receive status changes as push events. 

{
  "event": "instance.ready",
  "created_at": "2026-03-17T14:10:11Z",
  "data": {
    "instance_id": "inst_92a8f4",
    "name": "prod-web-fr-01",
    "region": "fr-par-1",
    "public_ipv4": "51.15.204.77"
  },
  "signature": "sha256=..."
}

Verify the HMAC signature on every webhook before processing events.

Production best practices

  • Rotate API tokens every 90 days.
  • Apply a maximum 10-second client HTTP timeout.
  • Implement exponential retry only for 429 and 5xx.
  • Correlate every action with a request_id in your logs.
  • Enable automated backups when the instance is created.
